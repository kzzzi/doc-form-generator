# 문서 양식 생성 데모 (AX 솔루션)

회의록·공문·보고서·안내문 등 자주 쓰는 문서 양식에 내용을 입력하면 초안을 만들어주는 프로토타입입니다. 외부 AI API나 API 키 없이, 서버도 없이 **브라우저 안에서만** 동작합니다.

## 🔗 라이브 데모 (GitHub Pages)

**https://kzzzi.github.io/doc-form-generator/**

## 사용 흐름 (4단계)

1. **정보입력** — 문서 유형(회의록/공문/보고서/안내문) 선택 → 유형에 맞는 필수 정보 입력(제목, 날짜, 참석자 등) → 문체(공식적/간결/상세) 선택
2. **초안생성** — 입력한 정보를 바탕으로 섹션별 초안을 생성. 제목·본문을 자유롭게 수정 가능 (플레인한 흰색 레이아웃)
3. **미리보기** — 문서 유형에 맞는 실제 서식(제목, 수신/일시 등 메타 정보, 섹션 구분)이 적용된 완성본 미리보기
4. **다운로드** — DOCX / PDF로 내보내기

## 핵심 특징

- 외부 AI 없이 규칙 기반으로 동작합니다. 사용자가 입력한 내용을 문체 옵션에 따라 문장 끝맺음·항목화 방식만 바꿔 조립하며, 내용을 지어내지 않습니다.
- 회의록(일시/장소/참석자), 공문(수신/담당자, 맺음말), 보고서(작성자/결론), 안내문(대상/문의처) 등 유형별로 다른 필드와 서식을 사용합니다.
- DOCX/PDF 생성이 모두 브라우저에서 직접 이루어져 서버가 필요 없습니다.

## 로컬에서 실행

빌드 과정이 필요 없는 순수 정적 사이트입니다:

```bash
cd doc-form-generator/docs
npx serve .
```

## 폴더 구조

- `docs/index.html`, `docs/app.js`, `docs/style.css` — 4단계 위저드 UI, 문서 유형별 필드 스키마·생성 로직·서식 미리보기
- `docs/vendor/` — docx.js 9.7.1(DOCX 생성), pdfkit 0.19.1 standalone 번들(PDF 생성) — 모두 오픈소스, 자체 포함(CDN 의존 없음)
- `docs/fonts/`, `docs/vendor/Pretendard-*.woff2` — Pretendard(SIL OFL). 웹 화면은 Pretendard를, DOCX는 Windows 기본 폰트인 "맑은 고딕"을(별도 포함 없이 이름만 참조), PDF는 실제 임베드된 Pretendard를 사용합니다.
