# 문서 양식 생성 데모 (AX 솔루션)

출장보고서·휴가신청서·업무인수인계서·계획서·회의록·공문·보고서·안내문 등 8가지 업무 문서 양식에, 자유 텍스트 한 덩어리를 입력하면 유형을 자동 감지해 초안을 만들어주는 프로토타입입니다. 외부 AI API나 API 키 없이, 서버도 없이 **브라우저 안에서만** 동작합니다.

## 🔗 라이브 데모 (GitHub Pages)

**https://kzzzi.github.io/doc-form-generator/**

## 사용 흐름 (4단계)

1. **정보입력** — 문서 내용을 자유 텍스트로 입력하면 문서 유형을 자동 감지(직접 선택으로 덮어쓰기 가능) → 문체(공식적/간결/상세) 선택
2. **초안생성** — 감지된 유형에 맞게 항목별 초안을 생성. 제목·본문을 자유롭게 수정 가능 (플레인한 흰색 레이아웃)
3. **미리보기** — 문서 유형별 실제 서식(공문의 수신/제목 정보표+번호매김 본문+"끝.", 회의록의 개요표, 휴가신청서의 결재란 등)이 적용된 완성본 미리보기
4. **다운로드** — DOCX / PDF로 내보내기

## 핵심 특징

- 외부 AI 없이 규칙 기반으로 동작합니다: 유형 감지, 라벨(`제목: 값`)·키워드 단서 기반 항목 추출, 문체별 문장 변환 모두 정규식과 템플릿 조립입니다.
- 8개 유형마다 필수 항목·문체 관례·마무리 방식·서식이 다릅니다 (예: 회의록은 문체와 무관하게 "논의함/결정함" 기록체 고정, 휴가신청서 사유는 항상 1~2문장, 계획서 기대효과는 항상 2~4개 항목, 공문은 여러 줄이면 가/나/다로 세분화).
- 입력에서 채워지지 않은 핵심 항목(이름·날짜·장소·담당자·기한 등)은 빈칸으로 남기지 않고 바로 검토·수정할 수 있도록 예시 성격의 더미 값으로 채웁니다.
- DOCX/PDF 생성이 모두 브라우저에서 직접 이루어져 서버가 필요 없습니다.

## 로컬에서 실행

빌드 과정이 필요 없는 순수 정적 사이트입니다:

```bash
cd doc-form-generator/docs
npx serve .
```

## 폴더 구조

- `docs/index.html`, `docs/app.js`, `docs/style.css` — 4단계 위저드 UI, 문서 유형별 필드 스키마·생성 로직·서식 미리보기
- `docs/vendor/` — docx.js 9.7.1(DOCX 생성), pdfkit 0.19.1 standalone 번들(PDF 생성) — 모두 오픈소스, 자체 포함(CDN 의존 없음)
- `docs/fonts/`, `docs/vendor/Pretendard-*.woff2` — Pretendard(SIL OFL). 웹 화면은 Pretendard를, DOCX는 Windows 기본 폰트인 "맑은 고딕"을(별도 포함 없이 이름만 참조), PDF는 실제 임베드된 Pretendard를 사용합니다.
